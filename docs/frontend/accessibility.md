# accessibility

## v1 requirements

- Use semantic HTML.
- Use one `h1` per page.
- Keep heading order logical.
- Use descriptive link text.
- Keep navigation keyboard accessible.
- Use visible focus styles.
- Maintain strong text contrast.
- Keep pages readable without JavaScript.

## validation

Before launch:

1. Navigate with keyboard only.
2. Confirm focus is visible on links.
3. Confirm mobile layout has no horizontal scrolling.
4. Confirm headings describe page structure.
5. Confirm link text makes sense out of context.
