# product overview

## problem

Ayman Shams needs one credible public site that connects his work without turning HealPath, SiriusShield, or Codex Coding OS into the umbrella brand.

## solution

A lean personal GitHub Pages site that introduces Ayman, explains the main work areas, and sends readers to the relevant project or profile.

## target users

- Technical reviewers.
- Founders and partners.
- Healthcare operators.
- Medical insurance operations partners.
- LinkedIn readers.

## value proposition

- Clear personal umbrella identity.
- Serious project routing without a bloated landing page.
- Correct boundary between personal work, business work, and open-source work.
- Low maintenance static site.

## scope boundaries

In scope:
- Homepage.
- Codex Coding OS page.
- Healthcare operations page.
- Notes page.
- 404 page.

Out of scope:
- Backend.
- Database.
- Forms.
- Analytics.
- Custom domain for v1.
- Blog engine.
- Case studies with sensitive details.

## core workflows

1. Visitor lands on homepage.
2. Visitor identifies Ayman's work areas.
3. Visitor chooses a page or external profile link.
4. Visitor reaches Codex Coding OS, healthcare operations, notes, GitHub, or LinkedIn.
5. Ayman updates the site through static file edits and Git commits.

## risks

| Risk | Control |
|---|---|
| Brand confusion | Ayman Shams is the first-page identity |
| Overbuilding | Static HTML and CSS only |
| Sensitive healthcare detail | Portfolio-level wording only |
| OpenAI affiliation confusion | Required disclaimer on Codex Coding OS page |
