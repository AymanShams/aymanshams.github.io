# requirements

## MVP requirements

| ID | Requirement | Acceptance criteria |
|---|---|---|
| REQ-001 | Homepage identifies Ayman Shams as the umbrella | `index.html` has Ayman's name as the main identity and no competing umbrella brand |
| REQ-002 | Homepage routes to work areas | Navigation links to Codex Coding OS, healthcare operations, and notes |
| REQ-003 | Codex Coding OS page includes disclaimer | Exact approved OpenAI non-affiliation sentence appears on the page |
| REQ-004 | Healthcare operations page stays portfolio-level | No patient, client, payer, financial, or internal operating details |
| REQ-005 | Notes page starts as a simple writing index | Page lists themes without needing a blog engine |
| REQ-006 | Site is static | No backend, database, form, analytics, cookie, or third-party script |
| REQ-007 | Site is responsive | Pages are readable on desktop and mobile without horizontal scrolling |
| REQ-008 | Site is accessible enough for v1 | Semantic HTML, logical headings, descriptive links, visible focus states |
| REQ-009 | External links are verified | GitHub and LinkedIn links are checked before launch |
| REQ-010 | GitHub Pages publishes from root of `main` | Pages deployment uses approved source |
