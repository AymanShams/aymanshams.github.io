# frontend guidelines

## design direction

The site should feel serious, direct, and readable. It should look like a personal operating homepage for technical and healthcare reviewers, not a startup landing page.

## layout

Use four pages:

| Page | Layout |
|---|---|
| Homepage | Name, positioning line, work areas, current focus, profile links |
| Codex Coding OS | Project summary, disclaimer, workflow focus, GitHub link |
| Healthcare operations | Healthcare and insurance operations overview |
| Notes | Writing themes and future note index |

## visual rules

- Text-only v1.
- No hero image.
- No decorative gradients, blobs, or oversized marketing sections.
- No cards inside cards.
- No popup, banner, newsletter box, or chat widget.
- Use restrained color, strong contrast, and enough white space.
- Keep content readable on phones without horizontal scrolling.

## navigation

Primary navigation:

- Home.
- Codex Coding OS.
- Healthcare operations.
- Notes.

Footer links:

- GitHub, after URL verification.
- LinkedIn, after URL verification.
- Email omitted until a public email address is approved.

## content voice

Use plain, serious language.

Avoid:

- Inflated claims.
- Startup hype.
- Generic personal-brand phrases.
- Unsupported performance claims.
- Healthcare or insurance claims that imply regulated service details.

## accessibility

- Use semantic HTML elements: `header`, `main`, `nav`, `section`, `footer`.
- Use one `h1` per page.
- Keep heading order logical.
- Ensure links are descriptive.
- Ensure visible focus states.
- Maintain strong color contrast.
- Make pages readable without JavaScript.

## responsive behavior

- Content width should stay comfortable on large screens.
- Navigation may wrap on small screens.
- Work areas should stack on mobile.
- Text should never overlap or shrink below readable size.

## content model

| Content item | Required in v1 | Notes |
|---|---|---|
| Name | Yes | Ayman Shams |
| Positioning line | Yes | Short and serious |
| Work areas | Yes | Four compact areas |
| OpenAI disclaimer | Yes | Exact approved wording on Codex page |
| Project links | Yes where verified | Omit unverified links |
| Headshot | No | Text-only preference |
| Blog posts | No | Notes index can start with themes |
| Contact form | No | Avoid form handling |

## initial copy direction

Homepage positioning line:

> AI-assisted coding systems, healthcare operations, insurance operations, and practical notes on how work gets designed and run.

Codex Coding OS summary:

> Codex Coding OS is an open-source workflow project for disciplined AI-assisted coding. It focuses on project manifests, documentation gates, repo instructions, handoffs, review, and validation.

Healthcare operations summary:

> Work across digital health, primary care, chronic care management, medical insurance operations, process design, and operating discipline.
